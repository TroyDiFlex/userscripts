# 🛍️ Магазин скриптов Tampermonkey

Личный каталог пользовательских скриптов для Tampermonkey. Работает как обычный сайт, ничего ставить не нужно — только браузер. Скрипты ставятся в один клик, есть отдельный приватный раздел под паролем и админка для добавления новых скриптов.

Эта инструкция написана для человека, который никогда не работал с кодом. Делайте всё строго по шагам — и через 20 минут у вас будет работающий магазин.

---

## 🎯 Что вы получите

- **Главную страницу** со всеми публичными скриптами — её можно открыть и установить любой скрипт одной кнопкой.
- **Приватный раздел** под паролем — для скриптов, которые не должны видеть посторонние.
- **Админку** — окно, в котором вы добавляете новые скрипты, не открывая код. Просто перетаскиваете файл и заполняете название.

Всё это бесплатно и хостится на GitHub.

---

## 📦 Раздел 1. Создание репозитория на GitHub

> Репозиторий — это «папка» на GitHub, в которой будут лежать ваши скрипты и сайт.

1. Зайдите на **https://github.com**. Если у вас ещё нет аккаунта — нажмите **Sign up** справа вверху и зарегистрируйтесь (пригодится логин — запишите его).
2. После входа нажмите на **зелёную кнопку «New»** слева вверху (рядом со списком репозиториев). Или просто откройте ссылку **https://github.com/new**.
3. В поле **Repository name** напишите название латинскими буквами без пробелов. Рекомендую: `scripts`.
4. В блоке **Public / Private** оставьте **Public**. ⚠️ Это важно: если выбрать Private — кнопка «Установить» работать не будет.
5. **Снимите галочку** с пункта «Add a README file» (он не нужен, у нас уже есть).
6. Прокрутите вниз и нажмите **зелёную кнопку «Create repository»**.

Готово. Теперь у вас открылась страница пустого репозитория. Запомните адрес — он выглядит так: `https://github.com/ваш-логин/scripts`.

---

## 📤 Раздел 2. Загрузка файлов магазина в репозиторий

1. На странице пустого репозитория вы увидите блок с текстом «Quick setup». Под ним есть фраза **«uploading an existing file»** — нажмите на неё. Или просто откройте: `https://github.com/ваш-логин/scripts/upload/main`.
2. Откройте на своём компьютере папку с файлами этого проекта (`index.html`, `admin.html`, папка `assets` и т.д.).
3. **Выделите все файлы и папки** (Ctrl+A на Windows / Cmd+A на Mac) и **перетащите их прямо в окно браузера**, в зону «Drag files here…».
4. Подождите, пока все файлы загрузятся (внизу появится список с галочками).
5. Прокрутите вниз — увидите блок «Commit changes». 👉 Нажмите **зелёную кнопку «Commit changes»**.

Через несколько секунд страница обновится — теперь у вашего репозитория есть содержимое.

---

## 🌐 Раздел 3. Включение GitHub Pages (это и есть ваш сайт)

> GitHub Pages — бесплатный хостинг от GitHub. Делает из вашего репозитория настоящий сайт.

1. На странице репозитория вверху найдите **вкладку Settings** (значок шестерёнки, самая правая в верхнем меню).
2. На открывшейся странице слева есть длинное меню. Прокрутите его и найдите раздел **Pages** (под «Code and automation»).
3. В основной части страницы вы увидите блок **«Build and deployment»**:
   - В выпадающем списке **Source** выберите **Deploy from a branch**.
   - В появившемся блоке **Branch** левый выпадающий список переключите на **main**, а правый оставьте **/ (root)**.
   - 👉 Нажмите **Save**.
4. **Подождите 1–2 минуты**, потом обновите эту же страницу (F5).
5. Если всё прошло хорошо — вверху появится зелёная плашка: **«Your site is live at https://ваш-логин.github.io/scripts/»**.

🎉 По этой ссылке ваш магазин уже работает! Откройте её в новой вкладке — увидите карточки скриптов.

> ⚠️ Кнопка «Установить» пока ведёт на пустые файлы — мы загрузим настоящие скрипты через админку в Разделе 8.

## 🔑 Раздел 4. Создание токена доступа (PAT)

> Простыми словами: токен — это пароль, который вы дадите админке, чтобы она могла сохранять изменения в ваш репозиторий.

1. Откройте **https://github.com/settings/tokens?type=beta**. Вы попадёте на страницу «Fine-grained personal access tokens».
2. Справа вверху нажмите **зелёную кнопку «Generate new token»**.
3. **Token name** — напишите любое название, например `scripts-admin`. Просто чтобы потом не забыть, зачем он.
4. **Expiration** — срок действия. Для удобства выберите **No expiration** (никогда не истекает) или Custom и поставьте срок год.
5. **Repository access** — выберите **Only select repositories**. Появится выпадающий список — выберите там ваш репозиторий (`scripts`).
6. Прокрутите вниз до блока **Permissions → Repository permissions**. Это самое важное:
   - Найдите строку **Contents** → справа от неё в выпадающем списке выберите **Read and write**. ⚠️ Без этого ничего не будет работать.
   - Остальные разрешения **не трогайте** — пусть будут «No access».
7. Прокрутите в самый низ страницы. Нажмите **зелёную кнопку «Generate token»**.
8. ⚠️ **ВНИМАНИЕ.** На следующей странице GitHub покажет ваш токен — длинную строку, начинающуюся с `github_pat_`. **Сразу скопируйте её** (нажмите на кнопку копирования рядом). Второй раз GitHub этот токен НЕ покажет — если потеряете, придётся создавать новый.

Сохраните токен в надёжное место (например, в заметках).

## ⚙️ Раздел 5. Подключение вашего репозитория к коду магазина

> Магазин должен знать, откуда брать скрипты для кнопки «Установить». Сейчас в коде написано `GITHUB_USER` — заглушка. Меняем её на ваш логин.

1. Откройте ваш репозиторий на GitHub: `https://github.com/ваш-логин/scripts`.
2. Нажмите на папку **assets**, потом на папку **js**, потом на файл **common.js**.
3. Над содержимым файла справа вверху есть **значок карандашика** (✏️ — кнопка «Edit this file»). Нажмите на него.
4. В самом верху файла вы увидите такую строку:
   ```js
   export const REPO = { owner: 'GITHUB_USER', name: 'REPO_NAME' };
   ```
5. **Замените `GITHUB_USER`** на ваш логин (в кавычках), а **`REPO_NAME`** — на название репозитория. Пример:
   ```js
   export const REPO = { owner: 'ivanov', name: 'scripts' };
   ```
6. Прокрутите вниз. Появится блок **«Commit changes»** — 👉 нажмите **зелёную кнопку «Commit changes»** (можно дважды, во всплывающем окне тоже).
7. Подождите ~1 минуту, пока GitHub Pages перевыпустит сайт.

Теперь магазин знает, где искать ваши скрипты.

## 🛠 Раздел 6. Первый вход в админку

1. Откройте в браузере адрес: `https://ваш-логин.github.io/scripts/admin.html`.
2. Вы увидите окно входа с двумя полями.
3. В поле **«GitHub токен (PAT)»** вставьте токен из Раздела 4 (тот, что начинается с `github_pat_`).
4. В поле **«Репозиторий»** напишите ваш логин и название через слэш. Пример: `ivanov/scripts`.
5. Нажмите **«Войти»**.

Если в верхней части появилась **зелёная точка** и название вашего репозитория — всё работает. Можно пользоваться.

> Токен сохраняется только в вашем браузере. На другом компьютере или в режиме инкогнито придётся вводить заново.

## 🔒 Раздел 7. Смена пароля приватного магазина

> По умолчанию пароль приватного раздела — `changeme`. Обязательно поменяйте сразу.

1. В админке слева нажмите **«⚙️ Настройки»**.
2. В блоке **«Пароль приватного магазина»** в поле «Новый пароль» напишите ваш новый пароль.
3. Под полем появится длинная зелёная строка — это **«хеш»** (необратимый отпечаток) вашего пароля. Сам пароль никуда не сохраняется, в коде живёт только хеш.
4. 👉 Нажмите **«⚡ Записать в репозиторий автоматически»**.
5. Через секунду появится сообщение «Пароль обновлён». Подождите ~1 минуту, пока GitHub Pages пересоберётся. Готово.

> 💡 **Если автозапись не сработала** (например, что-то с правами токена) — рядом написана пошаговая запасная инструкция: нажать «Скопировать хеш», открыть файл `assets/js/private.js` на GitHub, заменить старый хеш на новый и нажать «Commit changes».

## ➕ Раздел 8. Добавление вашего первого скрипта

1. В админке слева нажмите **«📜 Скрипты»**.
2. Справа сверху нажмите **«+ Добавить скрипт»**.
3. Заполните поля:
   - **Название** — как будет видно в магазине.
   - **ID** — короткое имя латиницей через дефис (например `my-helper`). Менять потом нельзя.
   - **Описание** — 1-2 предложения о том, что скрипт делает.
   - **Категория** — выберите из списка.
   - **Видимость** — «Публичный» (виден всем) или «Приватный» (только под паролем).
   - **Версия**, **Иконка** (любой эмодзи), **Цвет акцента**, **Теги** (нажмите Enter после каждого).
4. **Файл скрипта** — перетащите ваш `.user.js` файл в окошко с пунктиром (или нажмите и выберите).
5. **Скриншоты** (необязательно) — так же перетащите картинки.
6. Нажмите **«💾 Сохранить»** внизу.

Через ~30 секунд скрипт появится в магазине. Поделитесь ссылкой `https://ваш-логин.github.io/scripts/` — и любой может его поставить.

## 🔄 Раздел 9. Автообновление скриптов (для продвинутых)

Чтобы Tampermonkey у ваших коллег сам обновлял скрипт, когда вы заливаете новую версию — в начале файла `.user.js` в блоке `==UserScript==` должны быть две строки:

```js
// @updateURL    https://cdn.jsdelivr.net/gh/ВАШ-ЛОГИН/scripts@main/scripts/папка/имя.user.js
// @downloadURL  https://cdn.jsdelivr.net/gh/ВАШ-ЛОГИН/scripts@main/scripts/папка/имя.user.js
```

Замените `ВАШ-ЛОГИН`, `scripts`, `папка` и `имя.user.js` на ваши реальные. Это делается один раз для каждого скрипта.

## ❓ Раздел 10. Что-то пошло не так

| Проблема | Что делать |
|---|---|
| **Открыл сайт — белая страница или ошибка** | Подождите 2 минуты после изменений и обновите страницу через Ctrl+F5 (Cmd+Shift+R на Mac). |
| **Карточки видны, но кнопка «Установить» открывает 404** | В админке ещё не загружены сами `.user.js` файлы. Раздел 8. |
| **Карточки не показываются, в консоли ошибка** | Проверьте, что в `assets/js/common.js` правильно заменили `GITHUB_USER/REPO_NAME` на ваш логин и название репо. Раздел 5. |
| **Админка пишет «401 Unauthorized»** | Токен истёк или у него нет прав. Создайте новый по Разделу 4, обязательно поставьте **Contents: Read and write**. |
| **Админка пишет «404 Not Found»** | Вы написали название репо с ошибкой при входе. Нажмите «Выйти» вверху справа и введите правильное. |
| **Кнопка «Установить» скачивает файл вместо открытия в Tampermonkey** | У того, кто открывает магазин, не установлено расширение Tampermonkey. Покажите ему баннер «Как начать» на главной. |
| **Изменения в репозитории не появляются в магазине** | GitHub Pages обновляется ~1 минуту. Подождите и нажмите Ctrl+F5. |

## 📁 Что лежит в каких файлах (на всякий случай)

- `index.html` — главная страница (публичные скрипты)
- `private.html` — приватный раздел
- `admin.html` — админка
- `catalog.json` — «база данных» магазина (создаётся и меняется автоматически)
- `assets/css/` — оформление
- `assets/js/common.js` — здесь меняется `REPO`
- `assets/js/private.js` — здесь живёт хеш пароля
- `scripts/` — папки с реальными `.user.js` файлами
- `images/scripts/` — скриншоты карточек

Удачи! 🚀
